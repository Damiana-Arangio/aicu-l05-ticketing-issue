# Materiali Studenti - Lezione 05

## Cosa Usi

| File | Uso |
| --- | --- |
| `consegna.md` | Istruzioni della lezione e micro-task. |
| `template/issue-create-ticket.md` | Template da compilare per la issue `create ticket`. |
| `template/prompt-critica-issue.md` | Prompt per far controllare la issue senza chiedere codice. |

## Obiettivo

Preparare una issue verificabile per il task:

```txt
Serve creare ticket dal supporto.
```

Come visto in aula, il punto non e' chiedere subito codice. Il punto e' rendere chiari scope, criteri e verifica prima di autorizzare una patch.

## Output

Alla fine devi avere:

- una issue compilata;
- massimo 3 acceptance criteria;
- 2 verifiche manuali;
- almeno un non-goal;
- almeno una domanda aperta o una decisione presa.
